# 정확성 게이트 체크리스트 (Accuracy Gate)

릴리스 노트의 **모든 항목**에 대해 점검한다. 하나라도 실패하면 보강하거나 제거한다.

## 태그될 커밋 기준
- [ ] 모든 불릿이 태그될 커밋(또는 `HEAD`)의 코드에 근거한다
- [ ] 작업 트리에만 있는 미커밋/WIP 기능을 노트에 넣지 않았다
- [ ] 다른 브랜치에만 있는 기능을 넣지 않았다
- [ ] 의심 기능은 `git show <tag>:<path>` vs `rg <path>`로 직접 비교 검증했다

## 코드 근거
- [ ] PRD 로드맵의 "할 예정" 항목을 광고하지 않았다
- [ ] 각 기능이 명령 핸들러/store/UI에서 사용자 도달 가능함을 확인했다
- [ ] stub / `TODO` / `unimplemented!()` / 주석 처리 코드를 넣지 않았다
- [ ] 기능 플래그가 off로 빌드되는 기능을 넣지 않았다

## 과장 금지
- [ ] 코드 근거 없는 주장(성능 수치, "엄청 빠른" 등)이 없다
- [ ] 확신이 없는 항목은 제외했다 (제외 사유 기록)

## 개인정보
- [ ] 실제 사용자 데이터(`source_text`, `translated_text` 등)를 노출하지 않았다
- [ ] 내부 절대 경로/토큰/키/내부 호스트명을 노출하지 않았다

## 이중 언어 정합
- [ ] `새로운 기능`과 `Changelog`의 항목 **수**가 같다
- [ ] 항목 **순서**가 같다
- [ ] 같은 번호끼리 **같은 의미**를 담는다 (한쪽에만 있는 디테일 없음)
