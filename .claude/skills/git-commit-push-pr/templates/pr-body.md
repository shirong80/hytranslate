# PR 제목·본문 양식 (한국어)

PR 제목과 본문은 **한국어**로 작성한다. 타겟 대비 커밋·diff를 종합해 채운다.

---

## 제목

- 70자 이내, 한국어.
- 여러 커밋이면 개별 메시지를 나열하지 말고 **핵심 변경을 종합한 한 줄**로.
- 예: `사용자 프로필 조회 API 추가`, `결제 모듈 N+1 쿼리 최적화`, `로그인 토큰 갱신 무한루프 수정`

---

## 본문 양식

```markdown
## 요약
- {변경의 핵심을 1~3개 bullet로}

## 변경사항
- {주요 변경 파일/모듈}: {무엇을 어떻게 바꿨는지}
- {주요 변경 파일/모듈}: {내용}

## 테스트 계획
- [ ] {검증 항목 1}
- [ ] {검증 항목 2}
```

---

## 작성 가이드

- **요약**: "왜/무엇을" 중심. 리뷰어가 맥락을 빠르게 잡도록.
- **변경사항**: "어디를/어떻게" 중심. 파일·모듈 단위로. 큰 PR이면 도메인별로 묶어 기술.
- **테스트 계획**: 리뷰어/작성자가 확인할 체크박스. 자동화 테스트가 있으면 명시, 수동 검증 항목도 포함.
- 커밋 메시지는 영어지만 PR 본문은 한국어다. 파일 경로·심볼·명령어는 백틱으로 원문 유지.

---

## 예시

```markdown
제목: 사용자 프로필 조회 API 추가

## 요약
- 사용자 프로필 조회 엔드포인트(`GET /users/{id}/profile`)를 추가했습니다.
- 공통 로그 포매터를 분리해 재사용성을 높였습니다.

## 변경사항
- `user/profile.controller`, `user/profile.service`: 프로필 조회 엔드포인트 및 서비스 로직 추가
- `common/logger`: 포매터를 별도 함수로 추출, 기존 호출부 교체
- `README.md`: 프로필 API 사용법 문서 추가

## 테스트 계획
- [ ] 프로필 조회 정상 응답(200) 확인
- [ ] 존재하지 않는 사용자 요청 시 404 확인
- [ ] 기존 로그 출력 포맷 회귀 없음 확인
```
