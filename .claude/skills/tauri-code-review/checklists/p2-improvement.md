# P2 — 개선 포인트 체크리스트

코드 품질·유지보수성·성능 제안. 팀 상황에 따라 우선순위 조정 가능.

## IPC 성능 (references/type-contracts-and-platform.md)
- [ ] 대용량·스트리밍 데이터에 tauri::ipc::Channel이나 raw payload를 쓰는가
- [ ] 무거운 작업이 메인 스레드/코어를 막아 UI 프리징을 유발하지 않는가
- [ ] 빈번한 소량 IPC를 배치로 묶을 여지가 없는가
- [ ] 불필요한 JSON 직렬화/대용량 복사를 줄일 수 있는가

## 에러 처리 일관성 (references/rust-code-quality.md)
- [ ] thiserror/anyhow로 에러 처리 방식이 일관적인가
- [ ] 커맨드 반환이 Result 패턴을 일관되게 따르는가

## 코드 품질
- [ ] 중복 코드, 과도한 함수 길이, 매직 값이 없는가
- [ ] 커맨드/이벤트/모듈 네이밍이 명확한가
- [ ] 레이어 책임이 적절히 분리됐는가 (커맨드 = 얇은 경계, 로직 = 서비스)

## 테스트
- [ ] 변경된 커맨드/로직에 대한 테스트가 있는가
- [ ] 입력 검증·경계값·에러 경로 엣지 케이스를 다루는가
- [ ] 기존 정상 동작을 고정하는 회귀 방지 테스트가 있는가 (변경이 닿는 공유 커맨드/계약/State 기준)
- [ ] IPC 계약(커맨드·이벤트)의 직렬화 호환성을 검증하는 테스트가 있는가
- [ ] 회귀 테스트가 없는 고위험 변경 지점에 대해 추가를 권고했는가

## 문서화
- [ ] 복잡한 IPC 흐름·권한 부여 의도에 주석이 있는가
- [ ] capability/scope를 왜 그 범위로 잡았는지 근거가 기록됐는가
- [ ] 플랫폼 분기의 이유가 설명돼 있는가
