# 맥락 수집 양식

리뷰 시작 시 아래 입력물을 순서대로 확보하고 정리한다. Tauri 리뷰는 diff만으로는 권한·CSP를 볼 수 없으므로 **보안 설정 파일 수집이 필수**다.

---

## 1. 계획문서

요청: "설계서·기능명세·이슈/티켓을 공유해 주세요."

정리 양식:
```
- 변경 목적: {해결 문제 또는 구현 기능}
- 영향 범위: {새 커맨드/이벤트, 권한 변경, 플러그인 추가, 윈도우 구성 변화}
- 신뢰 경계 변화: {웹뷰에 새로 노출되는 기능, 외부 입력 출처}
- 비기능 요구: {성능(대용량 IPC), 크로스플랫폼(데스크톱/모바일)}
```

---

## 2. 변경 코드 (diff)

요청: "Git diff 또는 PR diff를 공유해 주세요."

정리 양식:
```
- 변경 파일 분류:
  - Rust 커맨드/핸들러 (src-tauri/src/): {목록}
  - 프론트엔드 (src/): {목록}
  - 설정 (*.json, Cargo.toml): {목록}
  - 플러그인 관련: {목록}
- 새/변경 #[tauri::command]: {커맨드명 + 인자 타입}
- 이벤트 emit/listen 변경: {이름 + 페이로드}
```

---

## 3. 보안 설정 파일 (Tauri 필수)

요청: "아래 파일도 함께 공유해 주세요 — diff에 없어도 권한·CSP 판단에 필요합니다."

| 파일 | 점검 목적 | 수신 여부 |
|------|-----------|-----------|
| `src-tauri/capabilities/*.json` | capability 범위, default 권한, core/플러그인 권한, scope, remote | {수신 / 미수신} |
| `src-tauri/tauri.conf.json` | CSP, withGlobalTauri, assetProtocol, dangerous*, 업데이터, 윈도우 | {수신 / 미수신} |
| `src-tauri/Cargo.toml` | Tauri 버전, 플러그인, 의존성(cargo audit) | {수신 / 미수신} |

미수신 파일이 있으면 해당 영역을 "확인 필요"로 명시하고, 받은 범위에서만 단정한다.

---

## 4. 추가 맥락 (필요시에만)

- Tauri 버전 / 타깃 플랫폼 (데스크톱 / iOS / Android)
- 타입 생성 도구 사용 여부 (tauri-specta 등)
- 팀 코딩 컨벤션

---

## 수집 완료 기준

계획문서 + diff + 보안 설정 파일(또는 미수신 명시)이 확보되면 2단계(분석)로 이동한다. 보안 두 축(① 커맨드 입력 검증, ② 권한·스코프)을 먼저 점검한다.
